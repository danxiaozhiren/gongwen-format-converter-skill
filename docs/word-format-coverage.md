# Word 格式覆盖矩阵

本文定义 `gongwen-format-converter` 对 Word `.docx` 格式的当前覆盖边界。它是项目能力声明和后续开发验收的基准。

当前定位：面向公文/内部简报的安全保守格式化器，优先保证正文内容、段落顺序、表格文本和既有对象不被破坏。它不是完整 Word 排版引擎，也不承诺对 Word 全对象、全样式、全视觉效果做等价处理。

## 状态说明

| 状态 | 含义 |
| --- | --- |
| `supported` | 已有脚本会自动处理，且有明确报告字段或可验证输出。 |
| `partial` | 已能处理常见情况，但复杂文档或边界对象仍需人工确认。 |
| `diagnose_only` | 只识别和报告，不自动修改。 |
| `preserve_only` | 尽量保留原对象和原状态，不主动重写。 |
| `unsupported` | 当前脚本没有可靠处理能力。 |
| `needs_design` | 能力方向明确，但需要先设计安全策略和验收样例。 |

## 优先级建议

下一阶段优先增强“诊断和保留不破坏”，再扩大自动修改范围。

1. 建立最小 `.docx` 样例集：正文、表格、页眉页脚/页码、图片/印章、文本框/形状、自动编号、目录/域代码、批注/修订。
2. 扩展报告覆盖：更多独立部件哈希、自动编号重建策略、可选字段更新工作流和渲染后检查。
3. 设计高风险对象策略：图片/印章只在显式要求和可验证时修改；文本框、形状、域代码先只诊断。
4. 增强渲染验收：在 PDF smoke check 基础上加入页面截图、版心/行数和视觉差异比较，避免只验证 OOXML 字段而忽略实际显示。

## 对外表述边界

推荐表述：

- “覆盖常见公文/简报 Word 格式整理，并对复杂对象做诊断和保留。”
- “对页面、段落、文字、表格文字、页码字段等安全范围自动处理。”
- “对图片、印章、文本框、形状、批注、修订、目录、复杂域等默认不主动重写。”

避免表述：

- “完全处理 Word 格式。”
- “支持 Word 全格式自动转换。”
- “保证与模板视觉效果 100% 一致。”
