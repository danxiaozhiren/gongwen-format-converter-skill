## 一句话介绍

`gongwen-format-converter` 是一个面向公文、内部简报、会议纪要、工作报告等办公文档的格式整理 Skill。它的目标不是替用户“乱改内容”，而是在保留正文不变的前提下，帮助用户完成 Word 文档的格式诊断、规范化排版和模板复刻。

## 适用场景

很多办公材料真正耗时的地方不是写正文，而是反复调整格式：标题字号不统一、正文行距不一致、一级标题和二级标题格式混乱、页边距不符合习惯、内部简报模板难复刻、表格和图片是否需要保留也不好判断。

这个 Skill 适合以下场景：

- 已经写好一篇 `.docx`，只想按公文或内部简报习惯调整格式。
- 有一份 Markdown 或纯文本材料，需要生成一份可流转的 Word 文档。
- 不确定当前文档格式哪里不规范，希望先诊断再决定是否修改。
- 有一份内部范文，希望只学习格式，不暴露正文内容，再把新材料套成同款版式。
- 处理内部材料时，希望默认不复述敏感原文，只输出格式层面的检查和建议。

## 核心能力

### 1. 仅格式化模式

用户已经定稿正文时，Skill 只调整外观，不改文字、不润色、不补写。

处理范围包括：

- 页面设置：A4、页边距、版心参考、分页策略。
- 文档元素：标题、主送机关、正文、附件、落款、日期等。
- 字体字号：标题、正文、各级标题的字体、字号、颜色、加粗、下划线等。
- 段落版式：首行缩进、左右缩进、固定行距、段前段后、对齐方式。
- 标题层级：`一、`、`（一）`、`1.`、`（1）`。
- 表格、图片、印章等对象的保留和风险提示。

在正式公文场景中，Skill 会优先参考《党政机关公文处理工作条例》和 `GB/T 9704-2012 党政机关公文格式` 的格式范围；在内部简报场景中，则使用“参照公文版式的内部办公材料”预设，不把经验规则误称为官方标准。

### 2. 格式诊断模式

该模式只读不改，适合在修改前先做体检。它会扫描整篇文档当前的格式状态，并生成诊断报告。

诊断内容包括：

- 页面设置、页眉页脚、页码区域。
- 段落角色识别：主标题、期号、正文标题、正文、一级标题、二级标题、附件、落款、日期等。
- 字体、字号、颜色、加粗、斜体、下划线。
- 行距、缩进、段前段后、对齐方式。
- 同类段落是否存在多个样式变体。
- 表格数量、表格样式样本。
- 图片、印章、图表等内嵌对象数量和处理提示。
- 与 `formal` 正式公文预设或 `brief` 内部简报预设的差异。

这个模式很适合用于“先别动文件，告诉我哪里不规范”。

### 3. 模板复刻模式

模板复刻被设计成两阶段流程，避免一上来就错误套版。

第一阶段：提取模板格式清单。

- 读取模板页面设置。
- 提取标题、正文、各级标题等角色样式。
- 识别字体、字号、颜色、下划线、行距、缩进等。
- 如果提供目标文档，会对比目标文档中哪些格式角色模板没有覆盖。
- 不复述模板正文，不输出敏感文本。

第二阶段：确认后应用。

当模板没有覆盖目标文档中的某类段落或对象时，Skill 会询问用户选择：

1. 使用公文/内部简报推荐格式。
2. 保留目标文档原格式。
3. 用户指定自定义格式。

用户确认后，Skill 再生成套用模板后的 Word 文档。

## 使用方式

安装后可以显式触发：

```text
$gongwen-format-converter

模式：仅格式化
输入文件：D:\path\材料.docx
预设：formal
要求：保留原文，只调整标题、正文、一级标题、二级标题、行距、缩进和页边距。
```

格式诊断：

```text
$gongwen-format-converter

模式：格式诊断
输入文件：D:\path\材料.docx
要求：先不要生成新文档，完整诊断页面、页眉页脚、段落角色、字体字号、颜色/下划线、行距缩进、表格图片，以及与正式公文格式的差异。
```

模板复刻：

```text
$gongwen-format-converter

模式：模板复刻
模板文件：D:\path\模板.docx
目标文件：D:\path\新材料.docx
要求：先提取模板格式清单，列出模板覆盖了哪些格式、目标文档还有哪些格式没有覆盖。不要直接生成，等我确认后再应用。
```

## 创作过程

这个 Skill 的设计过程来自一个很具体的办公痛点：许多材料正文已经写好，但每次交付前仍然要花大量时间调整 Word 格式。最初我想做“公文写作助手”，但在实际讨论和测试中发现，更真实、更高频的问题是“内容不用改，格式很费时间”。

因此，Skill 的定位逐步从“帮写公文”收敛为“公文与内部简报格式整理”。在迭代中，我把能力拆成三个模式：仅格式化、格式诊断、模板复刻。随后又补充了官方格式范围清单，让正式公文的处理尽量参考《党政机关公文处理工作条例》和 `GB/T 9704-2012`，同时把内部简报明确标注为内部办公场景的版式预设，避免把经验样式误说成强制规范。

在模板复刻上，我一开始采用的是直接提取模板样式并应用到目标文档，但测试后发现这种方式容易忽略模板未覆盖的段落类型。于是改成“两阶段流程”：先提取模板格式清单并列出未覆盖项，再询问用户如何处理，最后才应用。这样更符合真实办公场景，也能减少内部模板被误用或过度复刻的问题。

## 隐私说明

本 Skill 默认将公文、简报、会议、经营、财务、人事等材料视为内部敏感内容。格式诊断和模板复刻报告默认不输出完整正文，而是使用段落索引、角色、样式摘要和哈希辅助定位问题。模板复刻只学习样式指纹，不复述模板正文。
