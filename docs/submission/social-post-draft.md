正文：

```text
参加「SOLO 技能创作大赛」时，我做了一个很贴近办公场景的 Skill：gongwen-format-converter。

它解决的不是“AI 帮我写正文”，而是另一个更常见的问题：文章已经写好了，但 Word 格式还要反复调。

标题字号、正文行距、首行缩进、一级标题、二级标题、页边距、表格、页码、内部简报模板……这些事情单独看都不难，但每次手工处理都很费时间。

所以这个 Skill 的原则很简单：
内容不改、不补写、不润色、不调整段落顺序，只处理已有内容和已有对象的 Word 格式。

它目前有三个模式：

1. 现文格式化：正文不改，只调整页面、字体、字号、行距、缩进、标题层级、表格文字等格式。
2. 现文格式诊断：不改文档，先扫描整篇文档当前格式，输出页面、段落、字体、颜色、下划线、行距、表格、图片、页眉页脚等诊断报告。
3. 现文模板套用：先提取模板格式清单，列出目标文档还有哪些已有格式没有覆盖，确认后再套用模板。

正式公文场景会参考《党政机关公文处理工作条例》和 GB/T 9704-2012；内部简报则按内部办公材料预设处理。

我特别加了几个保守设计：
- 默认不在报告里复述完整正文；
- 模板套用只学习格式，不暴露模板正文；
- 页码只有明确要求才新增；
- 表格结构只有明确要求才统一规范；
- 图片、印章、文本框这类对象默认诊断并保留，避免误动。

用 Skill 之前：
- 手工调标题、正文、行距、缩进；
- 不确定哪里不符合公文/简报习惯；
- 模板能看但不好安全套用；
- 又怕 AI 顺手改了正文。

用 Skill 之后：
- 先诊断格式问题；
- 再确认格式方案；
- 最后生成整理后的 Word 文档；
- 报告里还能看到正文是否保持不变。

GitHub 地址：
https://github.com/danxiaozhiren/gongwen-format-converter-skill

#SOLO 技能创作赛
```

## 短视频脚本

```text
开头 0-10 秒：
你有没有遇到过这种情况：文章内容已经写完了，但 Word 格式还要调半天？标题、字号、行距、缩进、页边距、表格、页码、内部模板，每次都很烦。

10-25 秒：
我参加 SOLO 技能创作大赛时，做了一个 Skill，叫 gongwen-format-converter。它不是帮你乱写正文，而是专门处理公文和内部简报的 Word 格式。

25-45 秒：
它有三个模式。第一个是现文格式化，内容不改，只调整 Word 版式。第二个是现文格式诊断，先告诉你当前文档有哪些格式问题。第三个是现文模板套用，先提取模板格式清单，确认缺失项后再应用到新文档。

45-65 秒：
这里演示一下。我给它一份文档，让它先做格式诊断。它会识别页面、页眉页脚、段落角色、字体字号、颜色、下划线、行距缩进、表格图片和对象状态。然后我再选择要不要按正式公文或内部简报格式调整。

65-80 秒：
它还有一个很重要的边界：不改正文、不补缺失公文要素。页码、表格结构这类会改变版式结构的动作，也需要用户明确要求。

80-90 秒：
如果是内部模板，它默认不复述模板正文，只学习格式。这对内部材料比较友好。

结尾 90-100 秒：
这个 Skill 已经放在 GitHub，欢迎试用和提建议。#SOLO 技能创作赛
```
